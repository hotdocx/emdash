I think the simplification works, but with one correction.

A fixed-base internal pullback is good:

```text
Pullback_catd_func [A B] (F : A → B)
  : Functor (Catd_cat B) (Catd_cat A)
```

with:

```text
fapp0 (Pullback_catd_func F) D
  = Pullback_catd D F
```

and its arrow action is exactly existing `Pullback_funcd`.

But the more ambitious type:

```text
Functor (Catd_cat B)
  (Functor_cat (Functor_cat A B) (Catd_cat A))
```

is only partially valid. Functoriality in `D : Catd B` is canonical. Functoriality in `F : A → B` is not canonical for arbitrary displayed `D`, because a transformation `F ⇒ G` does not by itself give a map `F^*D → G^*D` unless `D` has chosen transport/opfibration structure.

For `PredPi`, though, we do not need that bad direction. We need the case where `F` is the varying projection:

```text
π_H : Total_cat H → Z
```

The clean way to package that is not as an ordinary natural transformation, because the domain `Total_cat H` varies with `H`. It should be a displayed functor:

```text
I_Z := Catd_cat Z

Totald_catd Z : Catd I_Z
  fibre at H = Total_cat H

ConstZ_catd Z : Catd I_Z
  fibre at H = Z

Total_proj1_functord Z :
  Functord (Totald_catd Z) (ConstZ_catd Z)
```

fibrewise:

```text
Fibre_func (Total_proj1_functord Z) H
  = Total_proj1_func H
```

Then we need the genuinely relative internal pullback:

```text
Pullback_catdd_func [I] [A B : Catd I]
  (F : Functord A B)
  : Functor (Catdd_cat B) (Catdd_cat A)
```

where:

```text
Catdd_cat B := Pi_cat (Catd_catd B)
```

and objectwise:

```text
D : Catdd B
↦
(i ↦ Pullback_catd (D[i]) (F_i))
```

Its fibrewise arrow action should reduce to existing `Pullback_funcd`.

Then `PredPi` becomes a clean derived construction:

```text
I_Z := Catd_cat Z
A   := Totald_catd Z
B   := ConstZ_catd Z
π   := Total_proj1_functord Z

E_const : Catdd B
  H ↦ E

π_star_E : Catdd A
  := Pullback_catdd_func π E_const

Cat_const : Catdd A
  H ↦ CatCat_catd (Total_cat H)

D : Catdd A
  := Functor_catdd π_star_E Cat_const

PredPi_catd E : Catd I_Z
  := Pi_catdd A D
```

with intended fibre:

```text
Fibre_cat (PredPi_catd E) H
=
Pi_cat
  (Functor_catd
    (Pullback_catd E (Total_proj1_func H))
    (CatCat_catd (Total_cat H)))
```

So I would slightly change the target from `PredPi_func` to `PredPi_catd`. Then `homd_curry` can use:

```text
Pullback_catd (PredPi_catd E) Edge_catd_fam
```

instead of forcing a Cat-valued functor plus `Fibration_cov_catd`.

That is probably the cleanest reconciliation: no object-only `PredPi` rule, no ordinary natural transformation hack, and no need for functoriality in arbitrary pullback base maps. The core missing primitive is relative pullback under a displayed functor.
